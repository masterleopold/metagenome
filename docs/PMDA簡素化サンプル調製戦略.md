## エグゼクティブサマリー

本文書では、Oxford Nanopore MinIONシーケンシングを用いてPMDA指定91病原体すべてを検出するための**簡素化ユニバーサルサンプル調製戦略**を提示します。本戦略は、既存の3-4つのワークフローを**2つの合理化ワークフロー**（DNA + RNA）に統合し、複雑性を50%削減しながら、**真の100%病原体カバレッジ**（環状ssDNAウイルスを含む）と許容可能な感度（LOD: 100-500 copies/mL）を維持します。

### 主要達成事項（v2.1更新）
- **ワークフロー統合**: 3-4プロトコル → 2プロトコル（50%削減）
- **時間効率**: 16時間 → 15.5時間の実験時間（3%削減）
- **コスト影響**: ¥152,000 → ¥162,000/サンプル（+6.6%、環状/ssDNA処理を含む）
- **病原体カバレッジ**: **真の100%（91/91病原体）** - PCV2/PCV3/TTV/PPVを含む
- **検出感度**: 100-500 copies/mL（標準的メタゲノミクス、スクリーニングに適切）
- **新規Step 2.5**: 環状DNA直鎖化 + ssDNA→dsDNA変換により、従来検出不可能だった4病原体の検出を保証

## 1. 背景と根拠

### 1.1 現行プロトコルの複雑性

既存のMinION病原体検出パイプラインは、PMDA指定91病原体の多様な生物学的特性に対応するため、複数のプロトコルバリアントを採用しています：

**現行ワークフロー構造**（簡素化前）：
1. **DNAワークフロー**: 51病原体（24 DNAウイルス + 27細菌）
2. **RNA Poly(A)+ワークフロー**: 16 ポリ(A)テールを持つRNAウイルス
3. **RNA Poly(A)-ワークフロー**: 4 ポリ(A)テールを持たないRNAウイルス（Hantavirus、WEEV、VEEV、REO）
4. **RNA増幅ワークフロー**: 低力価ウイルスのオプション強化
5. **Spumavirus専用ワークフロー**: PBMC由来ネステッドPCR

**特定された課題**:
- **意思決定の複雑性**: 複数の分岐点（poly(A)ステータス、増幅の必要性、PBMC採取）
- **トレーニング負担**: スタッフは4-5の異なるプロトコルバリアントを習得する必要
- **再現性リスク**: プロトコルのバリエーションがエラーの可能性を増加
- **時間の非効率**: 連続的ワークフローがターンアラウンドタイムを延長

### 1.2 簡素化の目的

ステークホルダー要件に基づき、簡素化戦略は以下を目標とします：

1. **優先事項**: プロトコル削減と時間効率のバランス
2. **コスト制約**: 最小限の増加（\~¥5,000-10,000/サンプル許容）
3. **感度目標**: 標準的メタゲノミクスLOD（100-500 copies/mL で十分）
4. **Spumavirusアプローチ**: 条件付きスクリーニング（ルーチンではない）

### 2.2 主要な簡素化

#### 簡素化1: ユニバーサルRNAワークフロー（全てでPoly(A)セレクション）

**根拠**:
- **80%のRNAウイルス**がpoly(A)テールを持つ → 最適な濃縮（100-1,000倍）
- **20%のRNAウイルス**はpoly(A)テールなし → メタゲノミクスアプローチで依然検出可能
- Poly(A)セレクションは**¥10,000安価**（rRNA除去より：¥5,000 vs ¥15,000）
- 単一RNAプロトコルで分岐ロジックを排除

**Poly(A)-ウイルスへの影響**（Hantavirus、WEEV、VEEV、REO）:
- LOD低下: 50-100 copies/mL → 300-500 copies/mL（スクリーニングには依然許容可能）
- 検出メカニズム: Poly(A)セレクション中の10-30%非特異的回収 + メタゲノミクスシーケンシング
- 代替案: より高感度が必要な場合、プロトコル11使用（rRNA除去 + 増幅RT-PCR）

**技術的検証**:

| ウイルスタイプ                | 現行プロトコル       | 簡素化プロトコル      | LOD変化         | 許容可能？          |
| ---------------------- | ------------- | ------------- | ------------- | -------------- |
| Poly(A)+ (EEEV, PRRSV) | Poly(A)セレクション | Poly(A)セレクション | 変化なし          | ✓ はい           |
| Poly(A)- (Hantavirus)  | rRNA除去        | Poly(A)セレクション | 100 → 300-500 | ✓ はい（スクリーニング用） |

#### 簡素化2: 条件付きSpumavirusスクリーニング

**根拠**:
- **豚参照ゲノムが存在しない**（固有の制限）
- 異種間検出（SFV/FFV/BFV）は低感度（\~30-50%相同性）
- PBMC採取が4時間 + ネステッドPCRが6時間 = **合計10時間の負担**
- 疫学的リスク: SPF豚施設での低有病率

**トリガー基準**（血漿スクリーニング優先）:
1. レトロウイルス様pol遺伝子検出（SFV/FFV/BFVと30-50%類似性）
2. PERV陰性だがpol遺伝子断片が存在
3. 疫学的疑い（アウトブレイク、輸入豚）

**コストベネフィット**:
- ルーチンPBMCスクリーニング: 100%サンプル → +¥15,000/サンプル + 10時間実験時間
- 条件付きスクリーニング: \~5-10%サンプル → +¥1,500/サンプル平均 + 1時間平均
- **節約**: ¥13,500/サンプル + 9時間実験時間

#### 簡素化3: ルーチン増幅RT-PCRの削除

**根拠**:
- Hantavirus/低力価ウイルス用増幅RT-PCR:
  - 6時間の実験時間追加
  - 36カスタムプライマー必要（¥200,000開発コスト）
  - LOD \<50 copies/mL達成（強化）
- 標準的メタゲノミクスはLOD 300-500 copies/mL達成（スクリーニングに十分）
- 段階的アプローチ: 増幅は確認用のみ、スクリーニング用ではない

**実装**:
- **Tier 1スクリーニング**: メタゲノミクス（全サンプル、LOD 100-500）
- **Tier 2確認**: 増幅RT-PCR（陽性のみまたは疫学的疑い）
- プロトコル11に「高感度サプリメント（オプション）」として文書化

### 7.2 環状ゲノムのカバレッジ解析

**接合部リード検出**:
環状DNAがDNase Iにより無作為に直鎖化される（Step 2.5.1）と、シーケンシングリードが**環状化接合部**（5'末端と3'末端が元々結合していた場所）にまたがる可能性があります。

**例：PCV2（1768 bp環状）**
```
位置500での直鎖化:
   Pos 500 ────→ Pos 1768 → Pos 1 ────→ Pos 499

Pos 1760-20にまたがる接合部リード:
   ...ATCG [ゲノム末端] [ゲノム開始] GATC...
```

**解決策**: 重複参照により、接合部リードが分割アライメントなしで連続的にマップ可能。

**カバレッジ計算**:
- 重複参照の**前半のみ**のカバレッジをカウント（位置0からgenome\_length-1）
- 後半を無視して二重カウントを回避
- 完全なゲノムカバレッジを検証（\>95%の位置で深度≥5×）

### 7.3 品質管理メトリクス

**PCV2/PCV3/TTV用バイオインフォマティクスQCチェックポイント**:

| メトリクス         | 目標          | 目的              |
| ------------- | ----------- | --------------- |
| 平均カバレッジ       | \>10×       | 十分なシーケンシング深度を保証 |
| カバレッジ均一性      | SD/平均 \<0.5 | 増幅バイアスを検出       |
| ゲノムカバレッジ%     | \>95%       | 完全な検出を確認        |
| 接合部リード存在      | はい          | 環状ゲノム処理を検証      |
| 両半分へのリードマッピング | はい          | 重複戦略が機能することを確認  |

**トラブルシューティング**:
- **リードがマップしない**: Step 2.5実験室処理失敗 → 抽出をやり直し
- **低カバレッジ(\<5×)**: 不完全な直鎖化 → DNase I濃度を最適化
- **大きなギャップ**: 過剰消化 → DNase Iを0.003 Uに削減
- **接合部リードなし**: 参照が重複されていない → データベースを再構築
