## タイムライン比較

### 従来マルチワークフロー（簡素化前）

```
1日目（実験時間9時間）：
├─ 09:00-10:00: サンプル調製
├─ 10:00-13:00: DNA/RNA抽出
├─ 14:00-16:00: ホスト除去（DNA）
└─ 16:00-20:00: 決定：rRNAかpoly(A)か？ ← 分岐

2日目（実験時間7時間）：
├─ 09:00-13:00: RNA特異的調製（ウイルスタイプにより異なる）
├─ 13:00-17:00: ライブラリ調製
└─ オプション：増幅RT-PCR（+6時間） ← 分岐

合計：16-22時間実験時間 + 意思決定の複雑性
```

### 簡素化ユニバーサルワークフロー（プロトコル12）

```
1日目（実験時間9時間）：
├─ 09:00-10:00: サンプル調製
├─ 10:00-13:00: DNA/RNA抽出
├─ 14:00-16:00: ホスト除去（DNA）+ Poly(A)（RNA）← 並列処理
└─ 16:00-20:00: DNAライブラリ調製開始

2日目（実験時間4時間）：
├─ 09:00-13:00: cDNA → RNAライブラリ調製
├─ 13:00-14:00: 両ライブラリのQC
└─ 14:00: MinION充填

合計：13時間実験時間（19%時間節約、分岐なし）
```

## トラブルシューティングクイックリファレンス

### よくある問題と解決策

```
問題1：DNA/RNA収量が低い
└─► サンプル容量を10 mLに増加
└─► SUPERase•In濃度を確認（RNA）
└─► 抽出キットのロット番号を確認

問題2：ホスト混入が高い（>10%）
└─► MBD2-Fcタンパク質量を増加
└─► インキュベーション時間を15分に延長
└─► 豚ゲノムのメチル化状態を確認

問題3：ライブラリ品質が低い
└─► 試薬の有効期限を確認
└─► アダプター：DNA比を最適化
└─► AMPureビーズ洗浄回数を増加

問題4：シーケンシング収量が低い
└─► フローセルQCを確認（>800アクティブポア）
└─► ローディング濃度を確認
└─► DuplexモードでQ30向上を有効化

問題5：Poly(A)-ウイルスが検出されない
└─► 予想される：LOD 300-500 copies/mL（許容可能）
└─► より高感度が必要な場合：プロトコル11に切替え
└─► rRNA除去 + 増幅RT-PCR使用
```
