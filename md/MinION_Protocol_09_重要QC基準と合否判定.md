## 1. QC基準体系の概要

### 1.1 多層QCシステムの構造

**品質保証の階層的アプローチ**

```yaml
QC階層構造:

  Level 1: サンプル品質 (Pre-analytical QC)
    目的: 適切なサンプル確保
    タイミング: サンプル採取〜保存
    基準:
      - 溶血グレード
      - 凝固なし
      - 適切な抗凝固剤使用
      - 保存温度・期間

  Level 2: 核酸抽出品質 (Extraction QC)
    目的: 高品質DNA/RNA取得
    タイミング: 抽出完了時
    基準:
      - 収量 (ng)
      - 純度 (A260/A280, A260/A230)
      - 完全性 (RIN, DIN)
      - 残存DNA (RNA抽出時)

  Level 3: ライブラリ品質 (Library QC)
    目的: シーケンス可能ライブラリ
    タイミング: ライブラリ調製完了時
    基準:
      - 濃度・総量
      - サイズ分布 (N50)
      - アダプター付加率
      - ダイマー含量

  Level 4: シーケンス品質 (Sequencing QC)
    目的: 十分なデータ取得
    タイミング: シーケンスラン中・終了時
    基準:
      - アクティブポア数
      - スループット
      - Read長
      - Q score

  Level 5: データ品質 (Bioinformatics QC)
    目的: 解析可能データ
    タイミング: データ処理時
    基準:
      - マッピング率
      - カバレッジ均一性
      - エラー率
      - 病原体検出信頼性

各レベルの関係:
  Level 1失敗 → Level 2以降全て影響
  Level 2失敗 → Level 3以降影響
  Level 3失敗 → シーケンス不可
  Level 4失敗 → データ不足
  Level 5失敗 → 病原体検出不可
```

### 1.2 合否判定の原則

**PMDAガイドライン準拠の判定基準**

```yaml
判定カテゴリー:

  合格 (Pass):
    - 全ての必須基準を満たす
    - 推奨基準の80%以上を満たす
    - 逸脱なし
    → 次ステップへ進む

  条件付き合格 (Conditional Pass):
    - 必須基準は満たす
    - 推奨基準の50-80%を満たす
    - 軽微な逸脱あり (documented)
    → 注意事項記録し、次ステップへ進む
    → データ解釈時に考慮

  不合格 (Fail):
    - 必須基準を1つでも満たさない
    - または重大な逸脱
    → 再実施または新サンプル

  保留 (Hold):
    - 判定に必要な情報不足
    - 追加測定必要
    → 追加QC実施後に再判定

判定の記録:
  - 全ての判定根拠を記録
  - 条件付き合格の理由明記
  - 不合格の場合、原因分析実施
  - ALCOA+原則遵守
```

## 2. サンプル採取・保存QC基準

### 2.1 血液サンプル採取時QC

**採血〜血漿分離の品質基準**

```yaml
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
採血QC基準
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

必須基準:

  1. 採血部位:
     合格: 前大静脈または外頸静脈
     不合格: 末梢静脈、毛細血管

  2. 抗凝固剤:
     合格: EDTA-2K または EDTA-3K
     推奨濃度: 1.5-2.2 mg/mL血液
     不合格: ヘパリン、クエン酸塩 (cfDNA分解促進)

  3. 採血量:
     必須最小: 5 mL
     推奨: 10 mL
     最適: 15-20 mL (バックアップ用)

  4. 採血時間記録:
     必須: 採血日時を秒単位で記録
     理由: cfRNA半減期短い (数分〜数時間)

推奨基準:

  5. 真空採血管使用:
     推奨: Cell-Free DNA BCT (Streck) または
           PAXgene Blood ccfDNA Tube
     利点: 細胞溶解抑制、cfDNA保護

  6. 採血手技:
     - 21G以下の針使用 (溶血防止)
     - 陰圧過大避ける
     - チューブ緩やかに反転混和 (8-10回)

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
血漿分離QC基準
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

必須基準:

  1. 遠心分離タイミング:
     合格: 採血後30分以内
     推奨: 採血後15分以内
     不合格: 採血後1時間以上

  2. 遠心条件 (2段階):
     第1回遠心:
       設定: 1,500-1,600×g, 10分, 4°C
       目的: 細胞除去

     第2回遠心 (必須):
       設定: 16,000×g, 10分, 4°C
       目的: 細胞debris完全除去
       理由: 細胞由来DNAコンタミ防止

  3. 血漿外観評価:
     溶血グレード (目視):
       Grade 0: 透明〜淡黄色 (合格)
       Grade 1: 微かなピンク (合格)
       Grade 2: 明らかなピンク (条件付き合格)
       Grade 3: 赤色 (不合格)

     溶血定量 (オプション):
       Hb濃度測定 (540 nm吸光)
       合格: <0.2 g/L
       不合格: >0.5 g/L

  4. 凝固確認:
     合格: フィブリン塊なし
     不合格: 凝固塊検出
     → サンプル廃棄、再採血

推奨基準:

  5. 血漿量:
     1本目採血管から: ≥2.5 mL
     2本目採血管から: ≥2.5 mL (バックアップ)
     総量: ≥5 mL

  6. アリコート:
     推奨: 2.5 mL × 2本 (解析用 + バックアップ)
     容器: 15 mL conical tube (RNase/DNase-free)

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
保存QC基準
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

必須基準:

  1. 凍結時間:
     合格: 血漿分離後30分以内に-80°C
     推奨: 15分以内
     不合格: 2時間以上室温放置

  2. 凍結方法:
     合格: -80°C直接凍結 または
           ドライアイス/エタノール急速凍結
     不合格: -20°C凍結 (不十分)

  3. 保存温度:
     短期 (<1週間): -20°C許容
     長期 (>1週間): -80°C必須
     超長期 (>6ヶ月): -80°C + 液体窒素バックアップ

  4. 凍結融解回数:
     必須: 最大2回まで
     推奨: 1回のみ
     記録: 各融解日時を記録

推奨基準:

  5. ラベル記載:
     - Sample ID (バーコード推奨)
     - 採血日時
     - 凍結日時
     - 動物ID
     - 凍結融解回数

  6. 保存期間:
     推奨最大: 2年間
     理由: cfDNA/RNA長期保存での分解

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
記録事項 (ALCOA+準拠):

  必須記録:
    □ 採血日時 (YYYY-MM-DD HH:MM:SS)
    □ 採血者氏名・署名
    □ 採血部位
    □ 採血量
    □ 抗凝固剤種類・ロット番号
    □ 血漿分離日時
    □ 遠心条件 (RCF, 時間, 温度)
    □ 溶血グレード
    □ 凝固有無
    □ 血漿量
    □ 凍結日時
    □ 保存場所 (フリーザーID)
    □ 特記事項
```

## 3. 核酸抽出QC基準

### 3.1 cfDNA/cfRNA抽出QC

**Zymo Quick-cfDNA/cfRNA Kit使用時の基準**

```yaml
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
抽出収量QC基準
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

必須基準 (5 mL血漿使用時):

  DNA収量:
    必須最小: 10 ng (total)
    推奨: 50-100 ng
    最適: >100 ng

    濃度:
      必須最小: 2 ng/μL (5 μL溶出)
      推奨: 10-20 ng/μL
      最適: >20 ng/μL

  RNA収量:
    必須最小: 5 ng (total)
    推奨: 20-50 ng
    最適: >50 ng

    濃度:
      必須最小: 1 ng/μL (5 μL溶出)
      推奨: 5-10 ng/μL
      最適: >10 ng/μL

  収量が低い場合の対処:
    5-10 ng (DNA): 使用可能だが複数抽出プール推奨
    <5 ng (DNA): 新サンプルまたは血漿量増加

判定:
  合格: DNA ≥10 ng かつ RNA ≥5 ng
  条件付き: DNA 5-10 ng または RNA 2-5 ng
  不合格: DNA <5 ng または RNA <2 ng

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
純度QC基準
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

NanoDrop測定:

  A260/A280比:
    合格: 1.8-2.0
    条件付き: 1.6-1.8 または 2.0-2.2
    不合格: <1.6 または >2.2

    対処 (不合格):
      <1.6: タンパク質汚染
        → AMPure XP追加精製
      >2.2: RNA混入またはアルカリ変性
        → RNase処理 (DNA解析) または
          pH確認

  A260/A230比:
    合格: 2.0-2.2
    条件付き: 1.5-2.0 または 2.2-2.5
    不合格: <1.5

    対処 (不合格):
      <1.5: 塩類・有機溶媒汚染
        → エタノール沈殿
        → AMPure XP複数回精製

推奨基準:

  Inhibitor test (qPCR):
    方法: サンプルとスパイクコントロールのqPCR比較
    合格: ΔCt <1 (inhibition <50%)
    不合格: ΔCt >2 (inhibition >75%)

    対処:
      希釈 (1:2, 1:5)
      追加精製

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
完全性QC基準
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

DNA完全性:
  測定: Bioanalyzer/TapeStation

  DIN (DNA Integrity Number):
    最適: DIN >7
    良好: DIN 5-7
    許容: DIN 3-5
    要注意: DIN <3

  注意: cfDNAは本質的に断片化
    DIN 3-5でも正常範囲

  サイズ分布:
    典型的cfDNA: 150-200 bp ピーク (ヌクレオソームサイズ)
    + 長鎖画分 (>1 kb)

    合格パターン:
      - 150-200 bp主ピーク
      - >1 kb画分存在
      - Smoothな分布

    異常パターン:
      - >10 kb過剰: 細胞溶解DNA混入疑い
      - <150 bp過剰: 過度の分解

RNA完全性:
  測定: Bioanalyzer RNA Pico Kit

  RIN (RNA Integrity Number):
    注意: cfRNAは断片化しているため低RIN正常

  DV200 (>200 nt 画分%):
    最適: DV200 >70%
    良好: DV200 50-70%
    許容: DV200 30-50%
    要注意: DV200 <30%

    対処 (DV200 <30%):
      - 新しいサンプル推奨
      - またはRNA量2倍使用

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
DNase処理QC (RNA解析時)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

必須基準:

  残存DNA除去効率:
    測定: qPCR (Sus scrofa β-actin gene)

    合格: ΔCt >10 (処理前比)
          = DNA >1,000倍減少
          = 残存DNA <0.1%

    条件付き: ΔCt 7-10
              = DNA 100-1,000倍減少

    不合格: ΔCt <7
            = DNA除去不十分
            → 追加DNase処理

  RNA回収率:
    合格: >80% (処理前比)
    条件付き: 60-80%
    不合格: <60%
      → DNase条件再最適化

記録事項:
  □ 抽出日時
  □ 使用血漿量
  □ キットロット番号
  □ 溶出体積
  □ Qubit濃度 (DNA, RNA)
  □ NanoDrop A260/A280, A260/A230
  □ DIN/RIN/DV200
  □ DNase処理 (実施有無、qPCR結果)
  □ 判定結果
```

## 4. ライブラリ調製QC基準

### 4.1 DNAライブラリQC基準

**SQK-LSK114 ライブラリの合否判定**

```yaml
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
濃度・収量基準
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

必須基準:

  Qubit dsDNA HS測定:
    総量:
      必須最小: 100 ng
      推奨: 200-300 ng
      最適: >300 ng

    濃度:
      必須最小: 10 ng/μL
      推奨: 20-30 ng/μL
      最適: >30 ng/μL

    体積:
      調製量: 12 μL (標準)
      最小許容: 10 μL

  測定再現性:
    3連測定CV: <10%
    許容: CV 10-15%
    不合格: CV >15% (ライブラリ不均一)

判定:
  合格: 総量 ≥100 ng, 濃度 ≥10 ng/μL, CV <10%
  条件付き: 総量 80-100 ng または CV 10-15%
  不合格: 総量 <80 ng または CV >15%

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
サイズ分布基準
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

TapeStation D5000測定:

  Read N50:
    最適: >15 kb
    推奨: 10-15 kb
    許容: 8-10 kb
    要改善: 5-8 kb
    不合格: <5 kb

  アダプターダイマー (<300 bp):
    最適: <2%
    良好: 2-5%
    許容: 5-10%
    不合格: >10%

    対処 (>10%):
      AMPure XP追加精製 (0.4× ratio)
      または BluePippin size selection

  サイズ分布パターン:
    望ましい:
      - Broad distribution (1-20 kb)
      - >10 kb画分: >30%
      - >20 kb画分: >10%

    要注意:
      - Sharp peak (サイズ選択過剰)
      - <1 kb過剰 (断片化)

  DIN (DNA Integrity Number):
    最適: DIN >7
    良好: DIN 5-7
    許容: DIN 3-5

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Adapter ligation効率
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

qPCR測定 (オプション、推奨):
  方法: MinION adapter特異的プライマー

  Ligation効率:
    最適: >90%
    良好: 80-90%
    許容: 70-80%
    不合格: <70%

  Ct値 (10 ng template):
    最適: Ct 15-18
    良好: Ct 18-20
    要注意: Ct >20 (低ligation効率)

  融解曲線:
    合格: 単一ピーク (80-85°C)
    不合格: 複数ピーク (非特異的増幅)

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Molarity計算と判定
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

計算式:
  Molarity (fmol/μL) = (濃度 ng/μL × 10^6) / (N50 bp × 650)

  例: 25 ng/μL, N50 10 kb
    = (25 × 10^6) / (10,000 × 650)
    = 3.85 fmol/μL

  ローディング量 (75 fmol目標):
    必要体積 = 75 / 3.85 = 19.5 μL

判定:
  最適: Molarity 5-10 fmol/μL
        必要体積 8-15 μL
  良好: Molarity 3-5 fmol/μL
        必要体積 15-25 μL
  条件付き: Molarity 2-3 fmol/μL
            必要体積 25-37 μL (希釈不要ギリギリ)
  要希釈: Molarity >10 fmol/μL
          必要体積 <8 μL (ピペッティング誤差大)

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
総合判定フローチャート
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

### 4.2 cDNAライブラリQC基準

**RNA由来ライブラリの特殊考慮事項**

```yaml
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
cDNA特異的QC基準
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

基本基準:
  DNA ライブラリと同一
  (総量、濃度、N50、ダイマー)

追加基準:

  1. Conversion効率:
     元のRNA量から期待されるcDNA量:
       理論値: RNA量 × 0.8-1.0

     例: 50 ng RNA → 期待cDNA 40-50 ng

     判定:
       最適: >80% conversion
       良好: 60-80%
       許容: 40-60%
       要調査: <40%

  2. ストランド特異性 (重要):
     測定: 既知ウイルスRNAスパイクとシーケンス

     (+)鎖リード割合:
       最適: >95%
       良好: 90-95%
       許容: 80-90%
       不合格: <80% (ストランド情報喪失)

  3. rRNA contamination:
     許容: <20% (SSR使用時)
     要注意: >20%
       → 次回SSR量増加

  4. サイズ分布 (cDNA特有):
     N50目標: 5-10 kb (RNAは短い)
     最適: >8 kb
     良好: 5-8 kb
     許容: 3-5 kb

     注意: RNA由来のためDNAより短いのは正常

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
DNA残存確認QC (重要)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

目的: RNAライブラリにDNA混入がないか確認

qPCR測定:
  - RT(+) sample: cDNA合成あり
  - RT(-) sample: cDNA合成なし (RNAのみ)

  期待:
    RT(+): Ct 20-25
    RT(-): Ct >35 (検出限界以下)

  判定:
    合格: ΔCt(RT- - RT+) >10
          = RNA特異的増幅
    不合格: ΔCt <7
          = DNA混入あり
          → DNase追加処理
```

```yaml
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

```yaml
  承認されたプロトコル、SOP、仕様からの乖離

分類:

  Critical Deviation (重大逸脱):
    定義: データ完全性、安全性、規制遵守に影響
    例:
      - PMDA必須QC基準不合格
      - サンプル取り違え
      - 記録改ざん
      - 重要機器の未校正使用
    対応: 即座報告、データ使用停止、CAPA必須

  Major Deviation (主要逸脱):
    定義: データ品質に影響するが致命的でない

      - 推奨QC基準の複数不合格
      - プロトコル手順の重要ステップ省略
      - 機器メンテナンス遅延
    対応: 24時間以内報告、影響評価、CAPA推奨

  Minor Deviation (軽微逸脱):
    定義: データ品質への影響minimal
    例:
      - 推奨QC基準1つ不合格
      - タイミングの軽微なずれ (<10%)
      - 記録の軽微な誤記 (訂正済み)
    対応: 記録、監視継続
```

## 9. 根本原因分析 (RCA)

### 9.1 5 Whys法

**繰り返し「なぜ?」を問う**

```yaml
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
5 Whys分析例: cfDNA収量低下
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

問題: cfDNA収量が5 ng (期待50 ng)

Why 1: なぜcfDNA収量が低い?
  → AMPure精製後の回収率が20%だった

Why 2: なぜAMPure回収率が低い?
  → ビーズ洗浄時にエタノールが残存していた

Why 3: なぜエタノールが残存?
  → 乾燥時間が不足していた (3分のみ)

Why 4: なぜ乾燥時間が不足?
  → プロトコルに「3-5分」と記載、3分を選択

Why 5: なぜ3分を選択?
  → 時間短縮を優先した

根本原因:
  プロトコル記載が範囲 (3-5分) で曖昧
  + 操作者が最小時間を選択する傾向

Corrective Action:
  今回のサンプル: 5分乾燥で再実施

Preventive Action:
  1. SOP改訂: 乾燥時間を「5分」に固定
  2. 目視確認基準追加: 「湿り気が残る程度」
  3. トレーニング実施: 適切な乾燥状態の認識
```

**要因の体系的分析**

```yaml

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

                        Duplex率低下
                             ↑
        ─────────────────────┼─────────────────────
        │                    │                    │
      人的要因            材料要因              機器要因
        │                    │                    │
    ・ピペッティング      ・DNA断片化          ・サーマルサイクラー
      過剰 (シェアリング)  ・Adapter劣化          温度ずれ
    ・AMPure混合不十分    ・Ligase失活         ・MinIONポア劣化

      逸脱                                      (basecalling遅延)

        │                    │                    │

  最も影響大きい要因:
    1. DNA断片化 (材料)

  1. DNA抽出プロトコル改善 (断片化最小化)
```

### 10.2 Method Validation

**分析法バリデーション(PMDA対応)**

```yaml
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Validation Parameters
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. Accuracy (正確性):
   方法: 既知濃度のcontrol DNA使用
   実施: 3濃度 × 3反復
   判定基準: 回収率 80-120%

2. Precision (精度):
   a. Repeatability (同一日、同一操作者):
      実施: 6反復
      判定基準: CV <15%

   b. Intermediate precision (異日、異操作者):
      実施: 3日間 × 2操作者 × 2反復
      判定基準: CV <20%

3. Linearity (直線性):
   方法: 5濃度 (10-1,000 ng)
   判定基準: R² >0.95

4. Range (測定範囲):
   決定: Linearityとprecision確認範囲
   PMDA用途: 10-500 ng

5. LOD (Limit of Detection):
   方法: Blank + 3 SD
   目標: <10 copies/mL (病原体)

6. LOQ (Limit of Quantification):
   方法: Blank + 10 SD
   判定基準: CV <20% at LOQ
   目標: 10-50 copies/mL

7. Specificity (特異性):
   方法: Cross-reactivity test
   判定基準: 非標的<1%検出

8. Robustness (頑健性):
   方法: 小変動導入 (温度±2°C、時間±10%など)
   判定基準: QC基準内維持

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

## まとめ

本章では、MinIONメタゲノム解析の全ステップにおけるQC基準と合否判定について詳述しました。

**重要ポイント:**

1. **多層QCシステム**
   - サンプル→抽出→ライブラリ→シーケンス→データ
   - 各レベルで厳格な基準
   - PMDA要求を上回る品質保証

2. **明確な合否判定**
   - 必須基準 vs 推奨基準
   - 合格/条件付き合格/不合格
   - データ使用可否の明確化

3. **逸脱管理とCAPA**
   - 全逸脱の記録と分類
   - 根本原因分析 (5 Whys, Fishbone)
   - 是正・予防措置の実施と検証

4. **トラブルシューティング**
   - ステップ別対処フロー
   - 迅速な問題解決
   - 再発防止

5. **継続的改善**
   - QC trendモニタリング
   - 定期レビュー会議
   - Method validation

6. **ALCOA+準拠**
   - 全記録の完全性
   - トレーサビリティ
